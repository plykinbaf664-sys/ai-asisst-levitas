# Telegram AI-ассистент — demo MVP

Локальный Telegram-бот для постановки задач голосом. Бот работает через long
polling, хранит привязанные чаты и задачи в локальном JSON-файле.

## Требования

- Node.js 20.6 или новее
- тестовый Telegram-бот от `@BotFather`
- OpenAI API key

## Настройка

1. Скопируйте `.env.example` в `.env`.
2. Заполните `TELEGRAM_BOT_TOKEN` и `OPENAI_API_KEY`.
3. Рекомендуется заполнить `MANAGER_TELEGRAM_USER_ID`, чтобы команды принимались
   только от руководителя.
4. При необходимости задайте `MANAGER_DISPLAY_NAME=Юрий`.

Узнать свой Telegram user id можно через отдельного служебного бота либо
временно запустить MVP без `MANAGER_TELEGRAM_USER_ID`, отправить сообщение боту
и посмотреть `from.id` через Telegram Bot API. После этого ограничение следует
включить.

## Запуск

```powershell
Copy-Item .env.example .env
npm test
npm start
```

Установка npm-пакетов не требуется.

## Ручная проверка

1. Создайте отдельный тестовый групповой чат и добавьте туда бота.
2. От имени руководителя отправьте в группе `/bind test`.
3. Откройте личный чат с ботом и отправьте голосовое:
   `В тестовый чат: Маша, подготовь три варианта оффера до завтра пятнадцать
   ноль-ноль, это нужно для запуска рекламы`.
4. Бот должен отправить структурированную задачу в тестовую группу.
5. В личке отправьте `/tasks`.
6. Проверьте fallback:
   `/task test Маша, подготовь три варианта оффера до завтра 15:00 для запуска рекламы`.

Локальное состояние создаётся в `data/storage.json`. Chat ID не задаётся через
env или код и появляется только после `/bind`.

## Команды

- `/start` или `/menu` — показать меню режимов.
- `/auto` — включить автоматическое определение задачи или личного запроса.
- `/bind <alias>` — привязать текущую тестовую группу.
- `/register` — зарегистрировать участника группы для последующего тегания.
- `/task <alias> <текст>` — создать задачу из текста.
- `/tasks` — показать активные задачи.

## Режимы в личном чате

- `📝 Поставить задачу` — текст и голосовые обрабатываются как задачи; если чат
  не определён, используется привязанный `test`.
- `🤖 Личный ассистент` — запрос обрабатывается только в личке. Для актуальных
  внешних фактов OpenAI может использовать web search и вернуть ссылки.
- По умолчанию действует режим `auto`: явные поручения отправляются как задачи,
  остальные запросы остаются у личного ассистента.
- `📋 Активные задачи` показывает текущие задачи, `💬 Чаты` — привязанные чаты.

## Ограничения MVP

- один локально запущенный процесс;
- только статусы `in_progress`, без закрытия задач;
- JSON-файл вместо БД;
- нет webhook, deploy, документов, аналитики и web UI;
- long polling не работает, если для этого же бота уже установлен webhook;
- корректность относительных сроков зависит от `ASSISTANT_TIME_ZONE`;
- модель может попросить переформулировать неоднозначную голосовую команду.
